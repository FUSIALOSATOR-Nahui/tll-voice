# [TLL-Voice] Walkthrough - Развертывание локального TTS

Документ описывает процедуру развертывания, настройки и интеграции локального компонента синтеза речи (TTS) на базе Silero Models в рамках проекта TLL-Voice.

---

## 1. Системные требования и зависимости

Для функционирования локального TTS требуются следующие программные и аппаратные компоненты:

* **Процессор:** Архитектура x86_64 с поддержкой набора инструкций AVX2.
* **Операционная система:** Windows или Linux с установленным интерпретатором Python 3.10+.
* **Зависимости библиотеки Python:**
  * `torch >= 2.0` (необходимо для работы моделей версий v4 и v5).
  * `torchaudio` (требуется для совместимости структуры размещения моделей).
  * `omegaconf` (необходимо для разбора конфигурационных файлов моделей).
  * `numpy` (используется для преобразования тензоров PyTorch в плоские массивы данных).

---

## 2. Архитектура и параметры моделей Silero TTS

### Версия v5_5_ru
В проекте настроена последняя версия модели для русского языка — `v5_5_ru`. Её параметры:
* **Поддерживаемые голоса (Speakers):** `aidar`, `baya`, `kseniya`, `xenia`, `eugene`.
* **Поддерживаемые частоты дискретизации (Sample Rates):** `8000` Гц, `24000` Гц, `48000` Гц. Для обеспечения максимального качества звука по умолчанию задано значение `48000` Гц.
* **Встроенные языковые функции:** Автоматическое определение ударений (auto-stress), восстановление буквы "ё" (homographs) и интонационная поддержка вопросительных предложений.

## 3. Реализация компонента в проекте

Локальный движок синтеза реализован в модуле [core/tts.py](file:///C:/Users/dede/00_project/TLL-Voice/core/tts.py) посредством класса [LocalTTSEngine](file:///C:/Users/dede/00_project/TLL-Voice/core/tts.py#L5).

### Принцип инициализации
Инициализация класса [LocalTTSEngine](file:///C:/Users/dede/00_project/TLL-Voice/core/tts.py#L5) выполняет автоматическую загрузку модели через PyTorch Hub:
```python
self.model, _ = torch.hub.load(
    repo_or_dir='snakers4/silero-models',
    model='silero_tts',
    language=self.lang,
    speaker='v5_5_ru',
    trust_repo=True
)
```
Загруженный файл модели кэшируется в локальной директории пользователя `~/.cache/torch/hub/`. При последующих запусках скачивание не производится, модель считывается из локального кэша.

### Алгоритм синтеза и управления скоростью
Метод `synthesize` принимает исходный текст и коэффициент ускорения речи `speed`. 
Поскольку в методе `apply_tts` моделей Silero отсутствует встроенный параметр регулирования темпа речи, изменение скорости воспроизведения реализуется путем модификации возвращаемой частоты дискретизации (sample rate):

1. Выполняется генерация аудиоданных через `apply_tts` с базовой частотой дискретизации `self.sample_rate` (48000 Гц).
2. Выходной тензор преобразуется в массив NumPy: `audio_data = audio_tensor.numpy()`.
3. Рассчитывается целевая частота дискретизации для воспроизводящего устройства: `target_sample_rate = int(self.sample_rate * speed)`.
4. Полученные данные передаются плееру, который за счет повышенной частоты воспроизводит файл быстрее без потери высоты голоса.

## 4. Ограничения автономного режима (Standalone)

Если для работы системы требуется полное отсутствие сетевых запросов, механизм `torch.hub.load` заменяется на локальный импорт файла модели `.pt`:
1. Загрузка файла модели напрямую по адресу: `https://models.silero.ai/models/tts/ru/v5_ru.pt`.
2. Загрузка в память через `torch.package.PackageImporter`:
   ```python
   model = torch.package.PackageImporter(local_file_path).load_pickle("tts_models", "model")
   ```
