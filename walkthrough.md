# [TLL-Voice] Walkthrough - Версия 0.6.5: Резервный лезиз (Proxy Fallback) и Ленивое кэширование

В данном релизе версии **0.6.5** были реализованы две ключевые архитектурные задачи:
1. Отказоустойчивый механизм автоматического переключения транскрипции на резервный прокси-сервер (Revival Proxy) при сбоях первичного прямого API-канала Google Gemini.
2. Событийно-ориентированный узел ленивого локального кэширования аудиозаписей (`debug_dumps/`) при системных ошибках API и программных зацикливаниях модели (STT Loop Detection).

---

## Основные изменения

### 1. Ядро приложения (`core/`)
* **[gemini.py](file:///c:/Users/dede/00_project/TLL-Voice/core/gemini.py)**:
  * Класс [GeminiClient](file:///c:/Users/dede/00_project/TLL-Voice/core/gemini.py#L49) теперь инициализирует два независимых клиента: `self.client` (основной прямой канал) и `self.fallback_client` (резервный прокси-канал). Метод `transcribe()` сначала выполняет генерацию контента через основной канал. При возникновении исключения (таймаут, лимиты, 500/503) автоматически выполняется повторная попытка через прокси.
  * Для предотвращения длительного зависания при блокировках провайдером на уровне маршрутизации пакетов (blackhole), для первичного клиента установлен таймаут **10 000 мс** (10 секунд). Для резервного прокси-клиента установлен таймаут **30 000 мс** (30 секунд).
  * Реализован класс исключения `RepetitionDetectedError` и функция `detect_repetition(text: str) -> bool`. Функция производит легковесный n-граммный анализ текста для детекции непрерывных лексических повторений слов (от 4 раз подряд) или словосочетаний (от 3 раз подряд). Если детектор срабатывает, метод `transcribe()` выбрасывает `RepetitionDetectedError`.
* **[engine.py](file:///c:/Users/dede/00_project/TLL-Voice/core/engine.py)**:
  * Метод `_setup_gemini()` считывает параметры резервного канала (`fallback_api_key`, `fallback_api_host` or their env-analogues `GEMINI_FALLBACK_API_KEY`/`GEMINI_FALLBACK_API_HOST`) и конфигурирует клиент.
  * Добавлен метод `_dump_audio_to_disk(self, wav_bytes: bytes, prefix: str) -> None` для записи аудиоданных из оперативной памяти на жесткий диск в директорию `debug_dumps/`. Имя файла содержит уникальный временной штамп высокой точности.
  * Модифицирован метод `_process_audio` для перехвата исключений:
    * При перехвате `RepetitionDetectedError` вызывается `_dump_audio_to_disk(wav_bytes, prefix="loop")`, а в UI отправляется сообщение `"Обнаружено зацикливание!"`.
    * При перехвате общих ошибок (`Exception`) вызывается `_dump_audio_to_disk(wav_bytes, prefix="err")`, а в UI направляется детальный текст ошибки API.

### 2. Конфигурация клиента
* **[.env](file:///c:/Users/dede/00_project/TLL-Voice/.env)**: Настроена прямая авторизация к Google по умолчанию. Добавлены переменные для автоматического переключения на прокси при сбое основного канала.
* **[.gitignore](file:///c:/Users/dede/00_project/TLL-Voice/.gitignore)**: Добавлена директория `debug_dumps/` для изоляции аварийных аудиофайлов от системы контроля версий.

### 3. Тесты (`tests/`)
* **[test_proxy_config.py](file:///c:/Users/dede/00_project/TLL-Voice/tests/test_proxy_config.py)**: Расширен юнит-тестами для проверки корректности инициализации обоих клиентов и работы автоматического резервирования (включая проверку изоляции окружения).
* **[test_lazy_cache.py](file:///c:/Users/dede/00_project/TLL-Voice/tests/test_lazy_cache.py)**: Новый тестовый набор для проверки логики ленивого кэширования и детектора зацикливаний:
  * `test_no_dump_on_success` — штатный цикл не создает файлов на диске.
  * `test_dump_on_api_error` — при падении API создается файл `err_*.wav`.
  * `test_dump_on_repetition_detected` — при зацикливании генерируется `RepetitionDetectedError` и сохраняется файл `loop_*.wav`.
  * `test_detect_repetition_logic` — детальный тест n-граммного анализатора текста.
* **[test_integration_proxy.py](file:///c:/Users/dede/00_project/TLL-Voice/tests/test_integration_proxy.py)**: Выполняет сквозной интеграционный тест переключения при сбое.
