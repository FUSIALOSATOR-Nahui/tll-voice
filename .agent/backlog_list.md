### Интеллектуальный ИИ-калибратор микрофонов при первом запуске (Audio Calibration Wizard)
*   **Цель:** Исключить ручной подбор индексов устройств ввода («угадывание») при онбординге на Windows и особенно Linux.
*   **Механика реализации:**
    1.  **Пре-фильтрация:** Сканировать аудиоустройства системы через `sounddevice.query_devices()`. Отбрасывать виртуальные петли (loopbacks), мониторы выходов и устройства с 0 каналов ввода.
    2.  **Запись:** Запустить параллельный захват аудио (3 секунды) со всех оставшихся физических портов в блоках `try/except` (игнорируя те, что заняты другими процессами, например ALSA busy).
    3.  **Анализ ИИ (Параллельные запросы):** Отправить записи параллельно в Gemini API (`gemini-3.1-flash-lite`) с включенной опцией **Structured Outputs** (`response_mime_type="application/json"`).
    4.  **Response Schema (JSON):**
        ```json
        {
          "signal_quality": 0.85,    // Оценка разборчивости речи (0.0 - 1.0)
          "noise_level": "medium",  // Уровень шума: low / medium / high
          "has_clipping": false,    // Перегрузка тракта
          "comment": "Слышны искажения от Bluetooth-гарнитуры"
        }
        ```
    5.  **Логика выбора:** Скрипт анализирует JSON, выбирает порт с максимальным `signal_quality` и минимальным шумом, сохраняет индекс в `config.json` и выводит пользователю дружелюбное уведомление с оценкой качества.
*   **Критические крайние случаи:**
    - Все микрофоны заблокированы (ALSA busy) -> Выдать предупреждение и вернуть дефолтное устройство.
    - Нет активных микрофонов -> Завершить калибровку с ошибкой.
